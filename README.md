# Activity Day 2

A small practice project that uses a branch-and-pull-request workflow.

## What was completed
- Updated three activity files with a small text addition.
- Added a new `feature2` file in the activity folder.
- Prepared changes as reviewable pull requests.

## How this project is managed
- All changes are submitted from feature branches.
- Pull requests are reviewed before merge, with admin approval required.
- If merge conflicts appear, the branch is updated with latest `main`, conflicts are resolved, and the PR is refreshed.
- Short, focused commits are preferred to keep reviews simple and reliable.
